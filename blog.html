Question & Answer

1. What is the difference between map, filter, foreach and find functions?

-- The map function is used to apply a specific function to each element of an array or collection and create a new array with the result. Takes a function as an argument and returns a new array containing the transformed elements.

-- The filter function is used to create a new array containing all elements that meet a certain condition. Takes as argument a function that returns a boolean indicating whether the element should be included in the new array.

-- The forEach function is used to iterate over an array or collection and apply a specific function to each element. It returns nothing, but can be used for side effects such as logging, changing state, or calling other functions.

-- The find function is used to return the first element in an array that satisfies a certain condition. It takes as an argument a function that returns a boolean value indicating whether the element matches the condition.

2. What is the difference between localstorage, session storage and cookies?

-- LocalStorage is a web storage mechanism that allows you to store key-value pairs on the client side without expiration. Data stored in LocalStorage is persistent. H. It can also be used after closing and reopening the browser.

-- SessionStorage is similar to LocalStorage in that it allows client-side storage of key-value pairs, but is session-specific. This means that data stored in SessionStorage is only available during the current browser session and is lost when the browser is closed.

-- Cookies are small text files that websites store on the client side. A cookie is sent to the server with each request, allowing the server to store information about the user.

3. What is event loop and how does it work?

-- The event loop is a core component of the JavaScript runtime that manages code execution in a single-threaded environment. Handle events and callbacks and coordinate code execution.

-- However, if the event handler function contains asynchronous code. In the case of setTimeout or polling functions, the event loop delegates the execution of its code to the browser-provided web API and keeps checking the event queue for other events to process.
